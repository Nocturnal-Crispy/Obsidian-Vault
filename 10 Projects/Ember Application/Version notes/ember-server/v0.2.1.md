# Ember Server v0.2.1 Release Notes

**Release Date:** 2026-03-29

## 🎉 New Features

### Enhanced Communication
- **Mentions System**: Added comprehensive @mentions support with role-based permissions and notification handling
- **Message Reactions**: Implemented backend support for emoji reactions with database schema
- **Kick Members API**: Added member moderation endpoints with WebSocket event broadcasting

### Security & Authentication
- **TOTP 2FA Implementation**: Complete Time-based One-Time Password authentication system
- **Device Recovery**: Enhanced device recovery with 2FA backup codes endpoint
- **Permission System**: Comprehensive role-based access control throughout the application

### Signal Protocol Enhancements
- **E2EE Layer 2**: Complete Signal Protocol implementation with history key management
- **Sender Key Distribution**: Improved group messaging with sender key storage and distribution
- **Session Tracking**: Added Signal session tracking store and HTTP endpoints
- **Dual-Stack Migration**: Support for both legacy and Signal Protocol encryption

### Voice & Video Infrastructure
- **Screen Sharing**: Extended server support for multiple simultaneous screen shares
- **Voice Handlers Refactor**: Improved code organization and performance of voice features
- **SFU Integration**: Enhanced Selective Forwarding Unit communication and management

### API Improvements
- **Token Rotation**: Automatic token refresh system for enhanced security
- **CORS Enhancements**: Better support for Electron file:// origins and cross-origin requests
- **Input Validation**: Comprehensive input validation across all endpoints

## 🐛 Bug Fixes

### Critical Security Fixes
- **Password Length Capping**: Limited password length to 72 characters (LOW-4)
- **Rate Limiting**: Applied rate limiting to device recovery endpoint (CRIT-5)
- **CORS Middleware**: Fixed CORS issues for Electron applications
- **Data Integrity**: Resolved database integrity issues and pagination problems

### Database & Storage
- **Unique Index Migration**: Added proper unique constraints for ember_keys table
- **Conflict Resolution**: Improved ON CONFLICT handling for peer-box operations
- **History Key Storage**: Enhanced storage and retrieval of encryption keys

### API & Endpoint Fixes
- **DM Collision Detection**: Fixed direct message collision detection issues
- **WebSocket Notifications**: Added real-time WebSocket notifications for DM events
- **Avatar Broadcasting**: Generic member update broadcasting for profile changes

### Performance & Reliability
- **Voice Handlers**: Improved security and performance of voice communication
- **Database Queries**: Optimized database queries and connection handling
- **Memory Management**: Reduced memory leaks and improved resource cleanup

## 📝 Installation

```bash

docker-compose -f docker-compose.prod.yml up -d

# Binary deployment
wget https://github.com/your-org/ember-server/releases/download/v0.2.1/ember-server-linux-amd64
chmod +x ember-server-linux-amd64
./ember-server-linux-amd64
```

## ⚠️ Breaking Changes

- Database schema updates require migration (handled automatically)
- Some legacy API endpoints have been removed
- Minimum Go version requirement updated to 1.21
- Configuration file format changes for enhanced security

## 🛠️ Migration Guide

### From v0.2.0 to v0.2.1
1. Backup your database
2. Update configuration files (new 2FA settings)
3. Run migration scripts (included in deployment)
4. Restart services

### Configuration Updates
```yaml
# New 2FA settings
auth:
  totp:
    enabled: true
    issuer: "Ember Server"
    backup_codes_count: 10

# Enhanced security settings
security:
  rate_limiting:
    enabled: true
    requests_per_minute: 60
```
