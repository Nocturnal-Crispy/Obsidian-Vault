# Ember Client v0.1.14 Release Notes

**Release Date:** 2026-03-17  
**Git Tag:** v0.1.14

## 🚀 Features

### Direct Messaging Enhancements
- **feat(dm): fulfill pending key requests for all DM embers on startup** - Ensures all pending DM key requests are fulfilled when the application starts
- **feat(dm): device key enrollment** - Client-side enrollment and fulfillment for device key requests
- **feat(dm): optimistic render + WS echo dedup (BP-4, BP-5)** - Improves user experience with optimistic rendering and prevents duplicate WebSocket messages
- **feat(dm): per-device peer-boxes + WS subscription hardening** - Enhanced security with per-device peer-boxes and improved WebSocket subscription handling

## 🐛 Bug Fixes
- **fix(dm): implement peer-box key distribution and fix BP-2/BP-3** - Resolves key distribution issues and addresses bugs BP-2 and BP-3

## 📝 Summary
This release significantly improves direct messaging functionality with device enrollment, optimistic rendering, and enhanced key distribution. The client now handles pending key requests automatically and provides a smoother user experience with reduced latency.

## 🔄 Upgrade Instructions
- From v0.1.13: Download the new version from releases page
- No configuration changes needed
- Existing DM conversations will continue to work seamlessly
