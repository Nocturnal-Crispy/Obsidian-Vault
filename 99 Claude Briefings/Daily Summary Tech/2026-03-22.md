## Story 2: GlassWorm Supply-Chain Attack — 72 Malicious VSX Extensions, 9M+ Installs
**Heat:** High | **Momentum:** RISING | **Source Tier:** 1 (Confirmed)
**Categories:** Security

The GlassWorm campaign has expanded significantly since initial reports. Attackers published 72 malicious Open VSX extensions mimicking popular developer tools — including linters, formatters, and AI-powered coding assistants like Claude Code and Google Antigravity. The extensions abuse `extensionPack` and `extensionDependencies` manifest fields to transform benign extensions into transitive delivery vehicles for malicious loaders.

The campaign has achieved over 9 million installs and affected at least 151 GitHub repositories between March 3–9. The loader uses invisible Unicode characters and the Solana blockchain as a dead-drop resolver to dynamically retrieve C2 server addresses. Objectives include credential theft, secret exfiltration, cryptocurrency wallet draining, and proxy conscription of developer endpoints.

**Why it matters:** This is the most sophisticated developer supply-chain attack of 2026 so far. The use of blockchain-based C2 resolution and transitive dependency abuse represents a new escalation in supply-chain attack techniques. Any developer using Open VSX extensions should audit their installed extensions immediately.

## Story 3: Oracle Critical RCE — CVE-2026-21992 (CVSS 9.8)
**Heat:** Medium | **Momentum:** NEW | **Source Tier:** 1 (Confirmed)
**Categories:** Security

Oracle released emergency security updates for CVE-2026-21992, a critical vulnerability (CVSS 9.8) in Oracle Identity Manager and Web Services Manager. The flaw enables unauthenticated remote code execution. Oracle confirmed it is "remotely exploitable without authentication."

**Why it matters:** Any organization running Oracle Identity Manager should patch immediately. Unauthenticated RCE in an identity management system is a critical attack surface for privilege escalation and lateral movement.

## Story 5: White House AI Legislative Framework — Federal Preemption Push
**Heat:** High | **Momentum:** SUSTAINED | **Source Tier:** 1 (Confirmed)
**Categories:** AI, Politics (cross-linked to [[99 Claude Briefings/Daily Summary Politics/2026-03-22]])

The White House's national AI legislative framework, released March 20, is receiving closer analysis. Key provisions include: sharp limits on legal liability for AI developers, federal preemption of state AI laws deemed too burdensome, standardized data center permitting, and protections against AI-generated content targeting children.

The preemption clause is the most consequential — it would block states like California from enacting stricter AI safety requirements. The administration aims to codify this into legislation by year-end.

**Why it matters:** This framework, combined with the Anthropic DOD situation, reveals a coherent federal strategy: reduce regulatory friction for AI companies that cooperate with defense objectives while punishing those that don't.
