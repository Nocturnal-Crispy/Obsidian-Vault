_Day One · Senior Cloud Consultant_

## 01 Project Scope & Objectives
- [ ] Define the primary business goal driving this cloud initiative
    - _Cost reduction? Scalability? Modernisation? Disaster recovery?_
- [ ] Clarify the project boundaries — what is explicitly in and out of scope
- [ ] Identify key deliverables and success criteria
- [ ] Establish a high-level timeline and key milestones
- [ ] Understand any regulatory or compliance drivers (GDPR, HIPAA, SOC 2, etc.)

## 02 Current State (As-Is Architecture)
- [ ] Inventory existing infrastructure — on-premises, co-lo, or existing cloud
    - _Servers, databases, storage, networking._
- [ ] Identify all applications in scope and their interdependencies
- [ ] Review current workload sizing (CPU, memory, storage, IOPS)
- [ ] Understand existing backup, DR, and high-availability setups
- [ ] Collect network diagrams and data flow documentation

## 03 Cloud Platform & Architecture
- [ ] Confirm target cloud provider(s) — AWS, Azure, GCP, multi-cloud, or hybrid
- [ ] Understand preference for IaaS / PaaS / SaaS / serverless
- [ ] Discuss migration strategy — lift-and-shift, re-platform, re-architect, or greenfield
- [ ] Identify landing zone / account structure requirements
- [ ] Clarify region and data residency requirements

## 04 Security & Compliance
- [ ] Identify data classification levels and sensitivity of workloads
- [ ] Review identity and access management (IAM) requirements and existing SSO/IdP
- [ ] Understand encryption requirements — in transit and at rest
- [ ] Determine network security requirements (VPN, private connectivity, WAF, DDoS)
- [ ] Review compliance certifications required from the cloud provider
- [ ] Clarify audit logging and monitoring obligations

## 05 Operations & Support Model
- [ ] Define operational ownership — client team, MSP, or shared responsibility
- [ ] Understand monitoring, alerting, and observability expectations
- [ ] Agree on RTO / RPO targets for each workload
- [ ] Identify incident management and on-call procedures
- [ ] Clarify patching, change management, and deployment cadences

## 06 Budget & Commercial
- [ ] Obtain approved project budget (CapEx and OpEx split)
- [ ] Discuss cloud spend targets and tagging/cost allocation strategy
- [ ] Understand existing cloud commitments (reserved instances, EDP, etc.)
- [ ] Identify financial governance and approval workflows
- [ ] Clarify billing contacts and preferred procurement channels

## 07 Stakeholders & Governance
- [ ] Map all key stakeholders — sponsors, technical leads, security, finance
- [ ] Agree on project governance structure and escalation path
- [ ] Establish preferred communication cadence and reporting format
- [ ] Identify any third-party vendors or partners involved
- [ ] Confirm sign-off process for architecture decisions and change requests
