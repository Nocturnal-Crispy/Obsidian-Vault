# Ember Client Release Notes

This directory contains release notes for all Ember Client versions.

## Structure

Each release note follows the naming convention: `vX.Y.Z.md` where:
- X = Major version (breaking changes)
- Y = Minor version (new features)  
- Z = Patch version (bug fixes)

## Template

Use the following template for new release notes:

```markdown
# Ember Client vX.Y.Z Release Notes

**Release Date:** YYYY-MM-DD  
**Git Tag:** vX.Y.Z

## 🚀 Features

### Feature Category
- **feat(scope): description** - Detailed explanation of the feature and its impact

## 🐛 Bug Fixes
- **fix(scope): description** - Bug that was fixed and its impact

## 🔧 Refactoring
- **refactor(scope): description** - Code improvements and optimizations

## 📝 Documentation
- **docs(scope): description** - Documentation updates

## 🧪 Testing
- **test(scope): description** - Test improvements or additions

## 📝 Summary
Brief summary of the release highlighting the most important changes and their impact on users.

## 🔄 Upgrade Instructions
- From vX.Y.Z-1: Download the new version from releases page
- Any configuration changes needed
- Migration steps if applicable
- Breaking changes (if any)

## 🏷️ Version Information
- Client Version: X.Y.Z
- Previous Version: X.Y.Z-1
- Total Commits: [number]
```

## Process

1. Create new release notes file using the template above
2. Update the version number and date
3. Fill in features, bug fixes, and other changes
4. Update summary and upgrade instructions
5. Use the file in the GitHub release creation process
